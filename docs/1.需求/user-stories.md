# GitPkg 用户故事

## 项目背景

GitPkg 是一款基于 .NET 10 AOT 的 GitHub Release 自动更新 CLI 工具。目标用户是**命令行工具终端用户**，通过全局 CLI 命令管理多个从 GitHub Release 分发的二进制工具。

## 产品定位

- **核心命令**: `gpkg install owner/repo` / `gpkg update`
- **更新策略**: 拉取最新 Release
- **包格式**: 通用二进制归档 (zip / tar.gz / tar.xz)
- **配置方式**: 清单文件 (JSON)
- **执行模式**: 手动触发

---

## Epic 1: 工具安装

### US-01: 从 GitHub Release 安装工具
**作为** 命令行用户
**我想** 通过 `gpkg install owner/repo` 安装一个 GitHub 上的工具
**以便** 快速获取并使用该工具，无需手动下载和配置

**验收标准**:
- 输入 `gpkg install <owner>/<repo>` 自动定位该仓库的最新 Release
- 自动根据当前操作系统和架构匹配对应的归档文件
- 若存在多个匹配资产，列出候选项让用户选择
- 下载归档 -> 解压 -> 放入安装目录
- 安装完成后打印工具路径和版本信息

### US-02: 指定安装版本
**作为** 命令行用户
**我想** 通过 `gpkg install owner/repo@v1.2.3` 安装指定版本
**以便** 在需要特定版本时不被最新版本覆盖

**验收标准**:
- 支持 `@<version>` 语法指定版本号或 tag
- 若指定版本不存在，给出明确错误提示

### US-03: 自定义安装目录
**作为** 命令行用户
**我想** 通过 `--dir` 参数指定工具的安装目录
**以便** 将不同工具安装到不同位置

**验收标准**:
- `gpkg install owner/repo --dir /custom/path` 安装到指定目录
- 默认安装目录为 `~/.gitpkg/tools/<name>/`

## Epic 2: 工具更新

### US-04: 更新单个工具
**作为** 命令行用户
**我想** 通过 `gpkg update <name>` 将指定工具更新到最新版本
**以便** 让我使用的工具保持最新

**验收标准**:
- 检查 GitHub Release 中是否有比当前安装版本更新的版本
- 有则下载、解压、替换旧文件
- 无则提示 "已是最新版本"
- 保留用户原有的配置文件（若有）

### US-05: 批量更新所有工具
**作为** 命令行用户
**我想** 通过 `gpkg update` 一次性更新所有已安装的工具
**以便** 不用逐个检查更新

**验收标准**:
- `gpkg update` (无参数) 遍历清单中所有工具
- 逐一检查更新并报告结果
- 最后打印汇总：更新了 N 个，已是最新 M 个，失败 K 个

### US-06: 检查更新（仅查看，不安装）
**作为** 命令行用户
**我想** 通过 `gpkg outdated` 查看哪些工具有新版本但暂不更新
**以便** 先了解变更内容再决定是否升级

**验收标准**:
- `gpkg outdated` 列出所有有新版本的工具
- 显示当前版本 -> 最新版本
- 不执行任何下载操作

---

## Epic 3: 包校验

### US-07: SHA256 完整性校验
**作为** 命令行用户
**我想** 在下载完成后自动校验归档的 SHA256
**以便** 确保下载的文件未被篡改或损坏

**验收标准**:
- 若 Release 中包含 `.sha256` 或 `checksums.txt` 文件，自动下载并校验
- 校验失败则丢弃已下载文件并报错
- 若无校验文件，给出警告但继续安装

### US-08: GPG 签名校验（可选）
**作为** 关注安全的用户
**我想** 通过 `--verify-gpg` 开启 GPG 签名校验
**以便** 验证发布者的身份

**验收标准**:
- 支持 `--verify-gpg <key-id>` 参数
- 校验失败则终止安装

## Epic 4: 环境管理

### US-09: 自动配置 PATH
**作为** 命令行用户
**我想** 在安装工具后自动将其可执行文件目录加入系统 PATH
**以便** 可以在终端任意位置直接调用该工具

**验收标准**:
- `gpkg install` 时通过 `--add-path` 自动修改 shell 配置文件
- 支持 bash / zsh / fish / PowerShell
- 给出需要重新加载 shell 的提示
- 卸载工具时通过 `gpkg uninstall` 清理 PATH 条目

### US-10: 卸载工具
**作为** 命令行用户
**我想** 通过 `gpkg uninstall <name>` 删除已安装的工具
**以便** 清理不再需要的软件

**验收标准**:
- 删除工具的安装目录
- 从清单文件中移除记录
- 提示检查 PATH 是否需要手动清理

---

## Epic 5: 清单管理

### US-11: 查看已安装工具列表
**作为** 命令行用户
**我想** 通过 `gpkg list` 查看所有已安装的工具及其版本
**以便** 了解当前环境中有哪些托管工具

**验收标准**:
- 列出工具名称、仓库来源、当前版本、安装时间
- 表格形式展示（通过 Spectre.Console）

### US-12: 清单文件结构
**作为** 命令行用户
**我想** 有一个 JSON 格式的清单文件记录所有已安装工具
**以便** 可以备份、分享或在其他机器上复现环境

**验收标准**:
- 清单文件默认路径: `~/.gitpkg/manifest.json`
- 格式包含: 工具名、仓库地址、安装版本、安装路径、安装时间
- 支持 `gpkg manifest export` 导出清单

### US-13: 从清单文件恢复
**作为** 命令行用户
**我想** 通过 `gpkg install --from manifest.json` 批量安装清单中的工具
**以便** 在新机器上快速搭建开发环境

**验收标准**:
- 读取清单文件并按配置逐工具安装
- 支持 `--dry-run` 预览将要安装的内容

---

## Epic 6: 搜索与发现

### US-14: 查看工具信息
**作为** 命令行用户
**我想** 通过 `gpkg info <name>` 查看工具的详细信息
**以便** 了解该工具在 GitHub 上的发布状态

**验收标准**:
- 显示: 仓库描述、最新 Release 版本、发布日期、可用资产列表
- 标注当前是否已安装以及安装的版本

## 非功能需求

| 需求 | 说明 |
|------|------|
| NFR-01 跨平台 | macOS / Windows / Linux 三平台支持，AOT 编译为单文件二进制 |
| NFR-02 性能 | 安装命令在 3 秒内完成启动 + 元数据获取（不含下载耗时） |
| NFR-03 体积 | AOT 编译后二进制 < 20MB |
| NFR-04 安全性 | 所有网络请求走 HTTPS；SHA256 校验默认开启 |

---

## 命令总览

```
gpkg install <owner/repo>[@version]  # 安装工具
gpkg update [name]                    # 更新工具（无参数则全部更新）
gpkg outdated                         # 检查更新但不安装
gpkg uninstall <name>                 # 卸载工具
gpkg list                             # 列出已安装工具
gpkg info <name>                      # 查看工具详情
gpkg manifest export                  # 导出清单
gpkg --version                        # 查看 GitPkg 自身版本
```
