## 1. 概述

GitPkg 是一款用于管理 GitHub Release 分发工具的包管理器。它自动处理下载、校验、解压和环境配置。

**核心能力**:
- 一行命令从 GitHub Release 安装工具
- 自动检测当前操作系统和架构，匹配正确的资产文件（压缩包或裸二进制）
- SHA256 完整性校验（默认开启）
- 多工具统一管理，支持批量更新
- AOT 编译为单文件二进制，无需运行时依赖

## 2. 安装 GitPkg

### 一键安装（推荐）

```bash
# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/wangbin1989/gitpkg/main/install.sh | bash

# 指定版本
curl -fsSL https://raw.githubusercontent.com/wangbin1989/gitpkg/main/install.sh | bash -s v2.1.0

# Windows (PowerShell)
Invoke-Expression (Invoke-RestMethod https://raw.githubusercontent.com/wangbin1989/gitpkg/main/install.ps1)
```

安装脚本会自动检测平台、下载最新版本并安装到 `~/.gitpkg/bin`。安装完成后会提示运行 `gitpkg init <shell>` 配置环境。

### 手动配置

如果跳过了自动配置，可以手动执行：

```bash
# zsh
eval "$(~/.gitpkg/bin/gitpkg init zsh)"              # 当前会话
eval "$(~/.gitpkg/bin/gitpkg init zsh)" >> ~/.zshrc   # 永久生效

# bash
eval "$(~/.gitpkg/bin/gitpkg init bash)"
eval "$(~/.gitpkg/bin/gitpkg init bash)" >> ~/.bashrc

# fish
~/.gitpkg/bin/gitpkg init fish | source
~/.gitpkg/bin/gitpkg init fish >> ~/.config/fish/config.fish
```

```powershell
# PowerShell
Invoke-Expression (& "$env:USERPROFILE\.gitpkg\bin\gitpkg.exe" init powershell)
```

如果只需要单独配置自动补全：

```bash
# zsh
eval "$(gitpkg completion zsh)"

# bash
eval "$(gitpkg completion bash)"

# fish
gitpkg completion fish > ~/.config/fish/completions/gitpkg.fish
```

```powershell
# PowerShell
Invoke-Expression (& "$env:USERPROFILE\.gitpkg\bin\gitpkg.exe" completion powershell)
```

### 手动安装

从 [GitHub Releases](https://github.com/wangbin1989/gitpkg/releases) 下载对应平台的归档文件，解压后将 `gitpkg` 放入 PATH 目录即可。

```bash
# macOS / Linux
tar -xzf gitpkg-v2.1.0-darwin-arm64.tar.gz
chmod +x gitpkg
mv gitpkg ~/.gitpkg/bin/

# Windows
# 解压 gitpkg-v2.1.0-windows-x86_64.zip，将 gitpkg.exe 放入 PATH 目录
```

### 验证安装

```bash
gitpkg --version
# 输出: gitpkg v2.1.0
```

## 3. 核心概念

### 工具名称

工具名称从仓库名自动提取。例如 `ggml-org/llama.cpp` 安装后的工具名为 `llama.cpp`。后续所有操作（更新、卸载、查询）都使用此名称。

### 清单文件

所有安装记录保存在 `~/.gitpkg/manifest.json`。可通过 `gitpkg list` 查看，或 `gitpkg manifest export` 导出。

### GITPKG_HOME 环境变量

`gitpkg init <shell>` 会设置 `GITPKG_HOME` 环境变量（默认为 `~/.gitpkg`），并将 `$GITPKG_HOME/bin` 加入 PATH。所有已安装工具的可执行文件通过符号链接集中在此目录。

### 文件结构

```
~/.gitpkg/
├── bin/                 # 可执行文件符号链接目录（链接名去除平台和架构信息）
│   ├── ripgrep -> ../tools/ripgrep/rg
│   ├── fd -> ../tools/fd/fd
│   └── gitpkg
├── manifest.json        # 清单文件
├── tools/               # 工具安装目录
│   ├── ripgrep/
│   ├── fd/
│   └── ...
└── tmp/                 # 下载临时目录
```

## 4. 命令详解

### 4.1 install — 安装工具

从 GitHub Release 下载并安装工具。

```bash
gitpkg install <owner/repo>[@<version>]
gitpkg install --from <manifest.json>
```

**示例**:

```bash
# 安装最新版本
gitpkg install BurntSushi/ripgrep

# 安装指定版本
gitpkg install sharkdp/fd@v10.0.0

# 从清单文件批量安装
gitpkg install --from tools.json
```

**选项**:

| 选项 | 说明 |
|------|------|
| `--from <file>` | 从清单文件批量安装 |

**安装流程**:

1. 解析仓库地址和版本号
2. 调用 GitHub API 获取 Release 信息
3. 根据当前 OS 和架构匹配可用资产
4. 若有多个匹配，列出选项交互选择
5. 下载资产文件（显示进度条）
6. SHA256 校验完整性
7. 解压到 `~/.gitpkg/tools/<name>/`（裸二进制文件直接复制）
8. 链接可执行文件到 `~/.gitpkg/bin/`
9. 更新清单文件
10. 输出安装成功信息

**效果示例**:

```
gitpkg install BurntSushi/ripgrep

下载 ripgrep-14.1.1-aarch64-apple-darwin.tar.gz ━━━━━━━━━━ 100%
校验 SHA256... ✓
解压到 ~/.gitpkg/tools/ripgrep...
  已链接 1 个可执行文件到 ~/.gitpkg/bin
✓ ripgrep 14.1.1 已安装到 ~/.gitpkg/tools/ripgrep
```

### 4.2 update — 更新工具

检查并更新已安装的工具到最新版本。

```bash
gitpkg update [<name>]
```

**示例**:

```bash
# 更新所有工具
gitpkg update

# 更新指定工具
gitpkg update ripgrep
```

**效果示例**:

```
gitpkg update

✓ ripgrep  v14.0.0 → v14.1.1
✓ fd       v9.0.0  → v10.0.0
= fzf      v0.55.0 (已是最新)
✗ bat      更新失败: 未找到适用于 macos/arm64 的资产

更新: 2 | 已是最新: 1 | 失败: 1
```

**特性**:
- 更新前将旧版本备份为 `.bak`，失败时自动恢复
- 更新完成后自动刷新 `~/.gitpkg/bin/` 中的符号链接

---

### 4.3 outdated — 检查更新

仅查看哪些工具有新版本，不执行下载。

```bash
gitpkg outdated
```

**示例**:

```
名称      当前版本    最新版本    仓库
─────────────────────────────────────────
ripgrep   v14.0.0    v14.1.1    BurntSushi/ripgrep
fd        v9.0.0     v10.0.0    sharkdp/fd

共 2 个工具可更新，使用 gitpkg update 更新
```

若全部为最新：

```
所有工具均为最新版本
```

### 4.4 uninstall — 卸载工具

删除已安装的工具。

```bash
gitpkg uninstall <name>
```

**示例**:

```bash
gitpkg uninstall ripgrep
```

**效果**:

```
卸载 ripgrep v14.1.1 (~/.gitpkg/tools/ripgrep)...
✓ ripgrep 已卸载
```

卸载时会自动清理 `~/.gitpkg/bin/` 中对应的符号链接。

### 4.7 init — 初始化 Shell

输出 shell 初始化脚本，用于 `eval` 集成。设置 `GITPKG_HOME` 环境变量并将 `~/.gitpkg/bin` 加入 PATH。

```bash
eval "$(gitpkg init <shell>"
```

**支持的 Shell**:

| Shell | 命令 |
|-------|------|
| zsh | `eval "$(gitpkg init zsh)"` |
| bash | `eval "$(gitpkg init bash)"` |
| fish | `gitpkg init fish \| source` |
| PowerShell | `Invoke-Expression (& 'gitpkg' init powershell)` |

**输出内容**（以 zsh 为例）:

```bash
# gitpkg shell init for zsh
export GITPKG_HOME='~/.gitpkg'
export PATH="$GITPKG_HOME/bin":$PATH
```

---

### 4.8 completion — 自动补全

输出 shell 自动补全脚本，用于 `eval` 集成。覆盖所有子命令和选项的补全。

```bash
eval "$(gitpkg completion <shell>)"
```

**支持的 Shell**:

| Shell | 命令 |
|-------|------|
| zsh | `eval "$(gitpkg completion zsh)"` |
| bash | `eval "$(gitpkg completion bash)"` |
| fish | `gitpkg completion fish > ~/.config/fish/completions/gitpkg.fish` |
| PowerShell | `Invoke-Expression (& 'gitpkg' completion powershell)` |

**补全覆盖范围**:

| 子命令 | 补全项 |
|--------|--------|
| (根) | 所有子命令名、`--help`、`--version` |
| `install` | `--from`、`--help` |
| `init` / `completion` | `zsh` `bash` `fish` `powershell` |
| `manifest` | `export`、`--help` |

### 4.9 manifest — 清单管理

```bash
# 导出清单
gitpkg manifest export                # 输出到 stdout
gitpkg manifest export > tools.json   # 保存到文件

# 从清单批量安装
gitpkg install --from tools.json
```

**清单格式**:

```json
{
  "version": 1,
  "tools": [
    {
      "name": "ripgrep",
      "repo": "BurntSushi/ripgrep",
      "version": "14.1.1",
      "installPath": "/home/user/.gitpkg/tools/ripgrep",
      "installedAt": "2026-06-05T10:30:00Z"
    }
  ]
}
```

### 4.10 self-update — 自更新

更新 GitPkg 自身到最新版本。

```bash
gitpkg self-update
```

**效果**:

```
当前版本: v2.0.2
最新版本: v2.1.0
发现新版本: v2.0.2 → v2.1.0
下载 gitpkg-v2.1.0-darwin-arm64.tar.gz (9.0 MB)...
校验 SHA256... ✓
解压...
替换二进制文件...
✓ GitPkg 已更新到 v2.1.0
```

**特性**:
- 自动匹配当前平台的归档文件
- SHA256 校验确保完整性
- Unix 下通过文件重命名原地替换
- Windows 下通过延迟批处理脚本绕过文件锁

## 5. 场景指南

### 场景一: 新机器快速搭建环境

```bash
# 1. 在旧机器导出清单
gitpkg manifest export > tools.json

# 2. 将 tools.json 复制到新机器

# 3. 在新机器执行安装
gitpkg install --from tools.json
```

### 场景二: 检查并升级所有工具

```bash
# 1. 查看哪些工具可更新
gitpkg outdated

# 2. 执行更新
gitpkg update
```

### 场景三: 安装特定版本用于调试

```bash
# 安装旧版本
gitpkg install sharkdp/fd@v9.0.0

# 调试完成后更新回最新
gitpkg update fd
```

### 场景四: 全局可用

安装工具后，可执行文件会自动链接到 `~/.gitpkg/bin/`。只需将该目录加入 PATH 一次：

```bash
# 初始化 Shell 环境（设置 GITPKG_HOME + PATH + 自动补全）
eval "$(gitpkg init zsh)" >> ~/.zshrc
eval "$(gitpkg completion zsh)" >> ~/.zshrc
source ~/.zshrc

# 现在可以在任意位置使用已安装的工具
rg --version
fd --version
```

## 6. 常见问题

### Q: 提示"未找到适用于 macos/arm64 的资产"？

Release 中没有匹配当前 OS 和架构的资产文件。检查该仓库的 Release 页面确认支持的平台，或联系仓库作者。

### Q: 支持只发布裸二进制文件的仓库吗？

支持。如果 Release 中没有压缩包，只有裸二进制文件（如 `tool-linux-amd64`），GitPkg 会自动识别并直接复制到安装目录，无需压缩包格式。

### Q: SHA256 校验失败？

下载的文件可能损坏。GitPkg 会自动删除损坏文件，请重新执行安装。

### Q: 能否管理私有仓库？

当前仅支持公开仓库。私有仓库需要 GitHub Token 认证，计划在后续版本支持。

### Q: 更新时会覆盖我的配置吗？

不会。GitPkg 更新前会备份旧目录，并跳过 LICENSE、README 等文档文件。用户创建的配置文件将被保留。

### Q: 卸载后 bin 目录中的链接会清理吗？

会。卸载时 GitPkg 会自动删除 `~/.gitpkg/bin/` 中对应的符号链接，无需手动干预。

### Q: GitHub API 速率限制？

未认证请求限制为 60 次/小时。大量操作时建议等待或使用 GitHub Token（后续版本支持）。

### Q: init 和 completion 有什么区别？

- `gitpkg init <shell>` — 设置 `GITPKG_HOME` 环境变量并将 `~/.gitpkg/bin` 加入 PATH
- `gitpkg completion <shell>` — 注册 shell 自动补全函数，支持 Tab 键补全子命令和选项

两者通常一起追加到 shell 配置文件中。
