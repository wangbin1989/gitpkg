## 1. 概述

GitPkg 是一款用于管理 GitHub Release 分发工具的包管理器。它自动处理下载、校验、解压和环境配置。

**核心能力**:
- 一行命令从 GitHub Release 安装工具
- 自动检测当前操作系统和架构，匹配正确的二进制归档
- SHA256 完整性校验（默认开启）
- 多工具统一管理，支持批量更新
- AOT 编译为单文件二进制，无需运行时依赖

## 2. 安装 GitPkg

### 从 Release 安装

访问 [GitHub Releases](https://github.com/ggml-org/gitpkg/releases)，下载对应平台的归档：

| 平台 | 文件名 |
|------|--------|
| macOS (Apple Silicon) | `gpkg-darwin-arm64.tar.gz` |
| Linux (x86\_64) | `gpkg-linux-x86_64.tar.gz` |
| Linux (ARM64) | `gpkg-linux-arm64.tar.gz` |
| Windows (x86\_64) | `gpkg-windows-x86_64.zip` |
| Windows (ARM64) | `gpkg-windows-arm64.zip` |

**macOS / Linux**:
```bash
tar -xzf gpkg-<platform>.tar.gz
chmod +x gpkg
sudo mv gpkg /usr/local/bin/
```

**Windows (PowerShell)**:
```powershell
Expand-Archive gpkg-windows-x86_64.zip
Move-Item gpkg.exe C:\Windows\System32\
```

### 验证安装

```bash
gpkg --version
# 输出: 1.0.0
```

## 3. 核心概念

### 工具名称

工具名称从仓库名自动提取。例如 `ggml-org/llama.cpp` 安装后的工具名为 `llama.cpp`。后续所有操作（更新、卸载、查询）都使用此名称。

### 清单文件

所有安装记录保存在 `~/.gitpkg/manifest.json`。可通过 `gpkg list` 查看，或 `gpkg manifest export` 导出。

### 文件结构

```
~/.gitpkg/
├── manifest.json        # 清单文件
├── tools/               # 工具安装目录
│   ├── ripgrep/
│   ├── fd/
│   └── ...
└── tmp/                 # 下载临时目录
```

## 4. 命令详解

### 4.1 install — 安装工具

从 GitHub Release 下载并安装工具。

```bash
gpkg install <owner/repo>[@<version>] [选项]
```

**示例**:

```bash
# 安装最新版本
gpkg install BurntSushi/ripgrep

# 安装指定版本
gpkg install sharkdp/fd@v10.0.0

# 自定义安装目录
gpkg install junegunn/fzf --dir /opt/tools

# 安装后自动加入 PATH
gpkg install BurntSushi/ripgrep --add-path

# 开启 GPG 签名校验
gpkg install sharkdp/fd --verify-gpg <key-id>
```

**选项**:

| 选项 | 说明 |
|------|------|
| `-d, --dir <path>` | 自定义安装目录，默认为 `~/.gitpkg/tools/<name>/` |
| `--add-path` | 安装后自动将工具目录加入 PATH |
| `--verify-gpg <key>` | 使用指定 GPG 密钥校验签名 |

**安装流程**:

1. 解析仓库地址和版本号
2. 调用 GitHub API 获取 Release 信息
3. 根据当前 OS 和架构匹配可用资产
4. 若有多个匹配，列出选项交互选择
5. 下载归档文件（显示进度条）
6. SHA256 校验完整性
7. 解压到目标目录
8. 更新清单文件
9. 输出安装成功信息

**效果示例**:

```
gpkg install BurntSushi/ripgrep

下载 ripgrep-14.1.1-aarch64-apple-darwin.tar.gz ━━━━━━━━━━ 100%
校验 SHA256... ✓
解压到 /Users/wangbin/.gitpkg/tools/ripgrep...
✓ ripgrep 14.1.1 已安装到 /Users/wangbin/.gitpkg/tools/ripgrep
```

### 4.2 update — 更新工具

检查并更新已安装的工具到最新版本。

```bash
gpkg update [<name>]
```

**示例**:

```bash
# 更新所有工具
gpkg update

# 更新指定工具
gpkg update ripgrep
```

**效果示例**:

```
gpkg update

✓ ripgrep  v14.0.0 → v14.1.1
✓ fd       v9.0.0  → v10.0.0
= fzf      v0.55.0 (已是最新)
✗ bat      更新失败: 未找到适用于 macos/arm64 的资产

更新: 2 | 已是最新: 1 | 失败: 1
```

**特性**:
- 更新前将旧版本备份为 `.bak`，失败时自动恢复
- 保留工具目录下的非二进制文件（如用户配置）

### 4.3 outdated — 检查更新

仅查看哪些工具有新版本，不执行下载。

```bash
gpkg outdated
```

**示例**:

```
名称      当前版本    最新版本    仓库
─────────────────────────────────────────
ripgrep   v14.0.0    v14.1.1    BurntSushi/ripgrep
fd        v9.0.0     v10.0.0    sharkdp/fd

共 2 个工具可更新，使用 gpkg update 更新
```

若全部为最新：

```
所有工具均为最新版本
```

### 4.4 uninstall — 卸载工具

### 4.7 manifest — 清单管理

```bash
# 导出清单
gpkg manifest export                # 输出到 stdout
gpkg manifest export > tools.json   # 保存到文件

# 从清单批量安装
gpkg install --from tools.json --dry-run   # 预览
gpkg install --from tools.json             # 执行安装
```

**清单格式**:

```json
{
  "version": 1,
  "tools": [
    {
      "name": "ripgrep",
      "repo": "BurntSushi/ripgrep",
      "version": "14.1.1",
      "installPath": "/home/user/.gitpkg/tools/ripgrep",
      "installedAt": "2026-06-05T10:30:00Z"
    }
  ]
}
```

## 5. 场景指南

### 场景一: 新机器快速搭建环境

```bash
# 1. 在旧机器导出清单
gpkg manifest export > tools.json

# 2. 将 tools.json 复制到新机器

# 3. 在新机器预览
gpkg install --from tools.json --dry-run

# 4. 执行安装
gpkg install --from tools.json
```

### 场景二: 检查并升级所有工具

```bash
# 1. 查看哪些工具可更新
gpkg outdated

# 2. 执行更新
gpkg update
```

### 场景三: 安装特定版本用于调试

```bash
# 安装旧版本
gpkg install sharkdp/fd@v9.0.0

# 调试完成后更新回最新
gpkg update fd
```

### 场景四: 全局可用

```bash
# 安装时自动加入 PATH
gpkg install BurntSushi/ripgrep --add-path

# 重新加载 shell 配置
source ~/.zshrc

# 现在可以在任意位置使用
rg --version
```

## 6. 常见问题

### Q: 提示"未找到适用于 macos/arm64 的资产"？

Release 中没有匹配当前 OS 和架构的归档文件。检查该仓库的 Release 页面确认支持的平台，或联系仓库作者。

### Q: SHA256 校验失败？

下载的归档文件可能损坏。GitPkg 会自动删除损坏文件，请重新执行安装。

### Q: 能否管理私有仓库？

当前仅支持公开仓库。私有仓库需要 GitHub Token 认证，计划在后续版本支持。

### Q: 更新时会覆盖我的配置吗？

不会。GitPkg 更新前会备份旧目录，并跳过 LICENSE、README 等文档文件。用户创建的配置文件将被保留。

### Q: GPG 校验需要什么条件？

系统需安装 `gpg` 命令行工具。未安装时 GitPkg 会给出明确提示。

### Q: 卸载后 PATH 还有残留？

卸载时 GitPkg 不会自动编辑 shell 配置文件（防止误删其他条目）。请手动检查 `~/.zshrc` 或 `~/.bashrc` 中的 `# Added by gitpkg` 注释块。

### Q: GitHub API 速率限制？

未认证请求限制为 60 次/小时。大量操作时建议等待或使用 GitHub Token（后续版本支持）。
