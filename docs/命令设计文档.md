# GitPkg 命令设计文档

**版本**: 2.2.0  
**日期**: 2026-06-23  
**状态**: 正式  

---

## 1. 概述

GitPkg 是一款基于 .NET 10 AOT 的 GitHub Release 自动更新工具。CLI 入口为 `gitpkg`，通过 System.CommandLine 注册 10 个子命令。

### 1.1 命令总览

| 命令 | 说明 | 参数 |
|------|------|------|
| `install` | 从 GitHub Release 安装工具 | `owner/repo[@version]` 或 `--from <file>` |
| `update` | 更新已安装的工具 | `[name]`（不指定则更新全部） |
| `outdated` | 检查已安装工具的更新 | 无 |
| `uninstall` | 卸载已安装的工具 | `name` |
| `list` | 列出已安装的工具 | 无 |
| `info` | 查看工具详情 | `name` 或 `owner/repo` |
| `init` | 输出 shell 初始化脚本 | `shell` |
| `completion` | 输出 shell 自动补全脚本 | `shell` |
| `manifest` | 清单管理 | 子命令 `export` |
| `self-update` | 更新 GitPkg 自身 | 无 |

### 1.2 全局选项

| 选项 | 说明 |
|------|------|
| `--help`, `-h` | 显示帮助信息 |
| `--version` | 显示版本号 |

## 2. 命令详解

### 2.1 install — 安装工具

```
gitpkg install <owner/repo[@version]>
gitpkg install --from <manifest.json>
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `owner/repo` | string | 与 `--from` 二选一 | GitHub 仓库，支持 `@version` 后缀指定版本 |
| `--from` | string | 与 `owner/repo` 二选一 | 从清单文件批量安装 |

**处理流程（单工具）**:

1. 解析 `owner/repo[@version]`
2. 调用 GitHub API 获取 Release（指定版本用 `/releases/tags/{tag}`，否则用 `/releases/latest`）
3. 获取当前平台信息（OS + Arch），匹配 Release 资产
4. 匹配到多个资产时交互选择，无匹配时从全部资产中手动选择
5. 下载资产到 `~/.gitpkg/tmp/`
6. SHA256 校验（若 Release 中存在校验文件）
7. 解压到 `~/.gitpkg/tools/{name}/`（裸二进制文件直接复制）
8. 链接可执行文件到 `~/.gitpkg/bin/`
9. 更新 `~/.gitpkg/manifest.json`

**处理流程（批量）**:

1. 读取清单文件（`File.ReadAllTextAsync` 一次性读入内存，避免文件占用）
2. 遍历清单中的工具，逐个执行单工具安装流程
3. 输出汇总：成功/失败数量

**错误处理**:

| 场景 | 行为 |
|------|------|
| 仓库不存在 | `✗ 资源不存在: ...` |
| 网络错误 | `✗ 网络错误: ...` |
| 清单文件不存在 | `清单文件不存在: ...` |
| 单个工具安装失败 | 记录错误，继续安装下一个 |

### 2.2 update — 更新工具

```
gitpkg update [<name>]
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 否 | 工具名称，不指定则更新全部 |

**处理流程**:

1. 加载清单文件，获取已安装工具列表
2. 若指定名称，按名称筛选；否则遍历全部
3. 对每个工具：
   a. 查询 GitHub API 获取最新 Release
   b. 比较版本号，相同则跳过（显示 `= name version (已是最新)`）
   c. 匹配平台资产并下载
   d. SHA256 校验
   e. 备份旧目录为 `.bak`，解压新版本（裸二进制文件直接复制）
   f. 解压/复制失败时自动恢复备份
   g. 刷新 `~/.gitpkg/bin/` 中的符号链接
   h. 更新清单中的版本信息
4. 输出汇总：`更新: N | 已是最新: M | 失败: K`

---

### 2.3 outdated — 检查更新

```
gitpkg outdated
```

**参数**: 无

**处理流程**:

1. 加载清单文件
2. 遍历每个工具，查询最新 Release 版本
3. 比较版本号，有更新则记录
4. 使用 Spectre.Console 表格输出：

```
名称      当前版本    最新版本    仓库
─────────────────────────────────────────
ripgrep   v14.0.0    v14.1.1    BurntSushi/ripgrep
```

5. 若全部为最新，输出 `所有工具均为最新版本`

---

### 2.4 uninstall — 卸载工具

```
gitpkg uninstall <name>
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 工具名称 |

**处理流程**:

1. 从清单查找工具，不存在则报错
2. 在删除目录前，记录已链接到 `~/.gitpkg/bin/` 的可执行文件名
3. 删除安装目录 `~/.gitpkg/tools/{name}/`
4. 清理 `~/.gitpkg/bin/` 中对应的符号链接
5. 从清单中移除记录

**输出**: `✓ name 已卸载`

---

### 2.5 list — 列出工具

```
gitpkg list
```

**参数**: 无

**处理流程**:

1. 加载清单文件
2. 使用 Spectre.Console 表格输出：

```
名称      版本      仓库                安装时间
───────────────────────────────────────────────────
ripgrep   v14.1.1   BurntSushi/ripgrep  2026-06-05
fd        v10.0.0   sharkdp/fd          2026-06-05
```

3. 清单为空时输出 `暂无已安装工具，使用 gitpkg install <owner/repo> 安装`

---

### 2.6 info — 查看详情

```
gitpkg info <name>
gitpkg info <owner/repo>
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 工具名称（已安装）或 `owner/repo`（任意仓库） |

**处理流程**:

1. 若输入匹配已安装工具名，从清单获取仓库信息
2. 若输入为 `owner/repo` 格式，直接使用
3. 查询仓库信息和最新 Release
4. 使用 Spectre.Console Panel 输出：

```
╭─ripgrep──────────────────────────────────────╮
│ 仓库:     BurntSushi/ripgrep                  │
│ 描述:     ripgrep recursively searches...     │
│ 已安装:   v14.1.1 (2026-06-05)                │
│ 最新版本: v14.1.1 (2026-06-01)                │
│ 可用资产:                                      │
│   - ripgrep-14.1.1-aarch64-apple-darwin.tar.gz│
╰──────────────────────────────────────────────╯
```

### 2.7 init — 初始化脚本

```
gitpkg init <shell>
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `shell` | string | 是 | 目标 shell：`zsh`、`bash`、`fish`、`powershell` |

**输出** — 根据 shell 类型输出不同的初始化脚本：

**zsh / bash**:
```bash
# gitpkg shell init for zsh
export GITPKG_HOME='~/.gitpkg'
export PATH="$GITPKG_HOME/bin":$PATH
```

**fish**:
```fish
# gitpkg shell init for fish
set -gx GITPKG_HOME '~/.gitpkg'
fish_add_path "$GITPKG_HOME/bin"
```

**PowerShell**:
```powershell
# gitpkg shell init for powershell
$env:GITPKG_HOME = '~/.gitpkg'
$env:Path = "$env:GITPKG_HOME\bin;" + $env:Path
```

### 2.8 completion — 自动补全脚本

```
gitpkg completion <shell>
```

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `shell` | string | 是 | 目标 shell：`zsh`、`bash`、`fish`、`powershell` |

**输出** — 使用静态定义的子命令和选项列表提供补全，覆盖所有子命令：

| 子命令 | 补全项 |
|--------|--------|
| (根) | `install` `update` `uninstall` `outdated` `list` `info` `init` `completion` `manifest` `self-update` `--help` `--version` |
| `install` | `--from` `--help` |
| `init` / `completion` | `zsh` `bash` `fish` `powershell` |
| `manifest` | `export` `--help` |
| 其他子命令 | `--help` |

**各 shell 注册机制**:

| Shell | 机制 |
|-------|------|
| zsh | `compdef` + `_describe` |
| bash | `complete -F` + `compgen` |
| fish | `complete -c` + `__fish_use_subcommand` |
| PowerShell | `Register-ArgumentCompleter` |

---

### 2.9 manifest — 清单管理

```
gitpkg manifest export
```

**子命令**:

| 子命令 | 说明 |
|--------|------|
| `export` | 将清单文件内容输出到 stdout |

**export 处理流程**:

1. 加载 `~/.gitpkg/manifest.json`
2. 序列化为 JSON 输出到 stdout
3. 可用 `> tools.json` 重定向到文件

---

### 2.10 self-update — 自更新

```
gitpkg self-update
```

**参数**: 无

**处理流程**:

1. 从程序集元数据获取当前版本号
2. 查询 GitPkg 仓库（`wangbin1989/gitpkg`）的最新 Release
3. 比较版本号，相同则跳过
4. 下载最新版本归档（带进度条）
5. SHA256 校验
6. 解压到临时目录
7. 替换当前二进制文件：
   - Unix：文件重命名替换（`File.Move`）
   - Windows：批处理脚本延迟替换（绕过文件锁）
8. 清理临时文件

## 3. 数据模型

### 3.1 清单文件 (`~/.gitpkg/manifest.json`)

```json
{
  "version": 1,
  "tools": [
    {
      "name": "ripgrep",
      "repo": "BurntSushi/ripgrep",
      "version": "v14.1.1",
      "installPath": "/Users/user/.gitpkg/tools/ripgrep",
      "installedAt": "2026-06-05T10:30:00Z"
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | int | 清单格式版本号，当前为 1 |
| `tools` | array | 已安装工具列表 |
| `tools[].name` | string | 工具名称（从仓库名提取） |
| `tools[].repo` | string | GitHub 仓库（`owner/repo`） |
| `tools[].version` | string | 已安装版本标签 |
| `tools[].installPath` | string | 本地安装目录绝对路径 |
| `tools[].installedAt` | datetime | 安装时间（UTC） |

### 3.2 目录结构

```
~/.gitpkg/
├── bin/                   # 可执行文件符号链接目录（PATH 只需包含此目录）
│   ├── rg -> ../tools/ripgrep/rg
│   ├── fd -> ../tools/fd/fd
│   └── gitpkg
├── manifest.json          # 已安装工具清单
├── tools/                 # 工具安装目录
│   ├── ripgrep/
│   └── fd/
└── tmp/                   # 下载临时目录
```

## 4. 错误处理策略

| 错误类型 | 处理方式 | 退出码 |
|----------|----------|--------|
| 仓库不存在 (404) | `✗ 资源不存在: ...` | 1 |
| 网络超时/连接失败 | `✗ 网络错误: ...` | 1 |
| SHA256 校验失败 | 删除损坏文件，抛出异常 | 1 |
| 工具未安装 | `✗ name 未安装` | 1 |
| 清单文件为空 | 提示并正常退出 | 0 |
| 操作取消 (Ctrl+C) | `操作已取消` | 1 |
| 其他异常 | `✗ 错误: message` | 1 |
